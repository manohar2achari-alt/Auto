Azure Client Secret Monitoring

Automate notifications for Azure App Registration secrets using Azure Automation and Managed Identity.

View Guide

Overview

Preventing Expiration Outages

Azure App Registration secrets have a hard expiration date. When they expire, applications immediately fail with 401 Unauthorized errors. This guide details how to automate notifications to ensure teams act in time.

The Strategy

Instead of manual tracking, we implement a tiered notification system. Emails are sent to the IAM Distribution List (DL) at key intervals: 60, 30, and 7 days before expiration.

Automation Engine

We use an Azure Automation Account with a System Assigned Managed Identity. This provides a secure, serverless environment to run daily compliance checks against your tenant.

How It Works

The solution involves a PowerShell runbook that executes daily. It audits all App Registrations and triggers alerts based on the "Days Remaining" until expiration.

Daily Audit
Check Expiry
Alert (60/30/7 Days)
  • 60 Days: Early warning email sent to IAM DL.
  • 30 Days: Second reminder to IAM DL.
  • 7 Days: Critical alert to IAM DL.

Step-by-Step Guide

1

Create Azure Automation Account

Go to the Azure Portal, search for "Automation Accounts", and create a new resource. Select your subscription and resource group. Ensure you enable System Assigned Managed Identity during creation or in the Identity blade afterwards.

2

Verify Managed Identity

Navigate to your new Automation Account. On the left menu, under "Account Settings", click Identity. Ensure the Status is set to On. Copy the "Object (principal) ID" as you will need it for the next step.

3

Assign Graph API Permissions

The Managed Identity requires permission to read application data. Since Graph permissions cannot be assigned via the Portal UI for Managed Identities, use the PowerShell script below. This grants Application.Read.All.

PowerShell 
$TenantID = "your-tenant-id"
$PrincipalID = "your-managed-identity-object-id"

# Assign Application.Read.All (Read-only access to apps)
New-MgServicePrincipalAppRoleAssignment -ServicePrincipalId $PrincipalID `
-PrincipalId $PrincipalID `
-ResourceId (Get-MgServicePrincipal -Filter "AppId eq '00000003-0000-0000-c000-000000000000'").Id `
-AppRoleId "9a5d68dd-52b0-4cc2-bd40-abcf44ac3a1c"
4

Create the Runbook

In the Automation Account, go to Runbooks and click "Create a runbook". Name it Monitor-AppSecrets, select PowerShell as the type, and use version 5.1.

5

Add Schedule

After pasting and publishing the script (found in the Scripts section below), go to Schedules in the Runbook menu. Create a schedule to run daily (e.g., at 6:00 AM) to ensure consistent monitoring.

Scripts

PowerShell Runbook: Monitor & Alert

This script checks for expiring secrets and sends email notifications to the IAM DL at 60, 30, and 7 day marks. It does NOT auto-renew secrets.

Runbook.ps1 
try {
    # Connect using Managed Identity
    Connect-MgGraph -Identity
    
    # Configuration
    $AlertDays = @(60, 30, 7)    # Trigger alerts on these exact days
    $IamDlEmail = "iam-dl@example.com"
    $SenderEmail = "automation-noreply@example.com"
    
    $Apps = Get-MgApplication -All
    
    foreach ($App in $Apps) {
        foreach ($Cred in $App.PasswordCredentials) {
            $ExpiryDate = [DateTime]$Cred.EndDateTime
            $DaysLeft = ($ExpiryDate - (Get-Date)).Days
            
            # --- Alerting Logic (60, 30, 7 Days) ---
            if ($DaysLeft -in $AlertDays) {
                Write-Output "Sending $DaysLeft-day alert for '$($App.DisplayName)' to IAM DL."
                
                $EmailBody = "The client secret for '$($App.DisplayName)' (App ID: $($App.AppId)) expires in $DaysLeft days on $ExpiryDate. Please rotate credentials immediately."
                
                $Params = @{
                    Message = @{
                        Subject = "ACTION REQUIRED: Secret for $($App.DisplayName) expires in $DaysLeft days"
                        Body = @{ ContentType = "Text"; Content = $EmailBody }
                        ToRecipients = @( @{ EmailAddress = @{ Address = $IamDlEmail } } )
                    }
                }
                # Uncomment to enable sending via Graph API:
                # Send-MgUserMail -UserId $SenderEmail -BodyParameter $Params
                
                # If using SMTP/Relay, replace the above with Send-MailMessage logic
            }
        }
    }
}
catch {
    Write-Error "An error occurred: $_"
}

Kusto Query (Monitoring)

Track your automation jobs in Azure Monitor to see when alerts are triggered.

Query.kql 
AzureDiagnostics 
| where ResourceProvider == "MICROSOFT.AUTOMATION" 
| where Category == "JobStreams" 
| where ResultDescription has "Sending" 
| project TimeGenerated, RunbookName_s, ResultDescription
| order by TimeGenerated desc
