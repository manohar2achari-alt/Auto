Azure Client Secret Lifecycle

Automate notifications and renewal for Azure App Registration secrets using Azure Automation and Managed Identity.

View Guide

Overview

Preventing Expiration Outages

Azure App Registration secrets have a hard expiration date. When they expire, applications immediately fail with 401 Unauthorized errors. This guide details how to automate notifications and renewal to ensure uptime.

The Strategy

Instead of manual tracking, we implement a tiered notification system. Emails are sent to the IAM Distribution List (DL) at key intervals: 60, 30, and 7 days before expiration.

Automation Engine

We use an Azure Automation Account with a System Assigned Managed Identity. This provides a secure, serverless environment to run daily compliance checks against your tenant.

How It Works

The solution involves a PowerShell runbook that executes daily. It audits all App Registrations and triggers actions based on the "Days Remaining" until expiration.

Daily Audit
Check Expiry
Alert (60/30/7 Days)
Auto-Renew (< 2 Days)
  • 60 Days: Early warning email sent to IAM DL.
  • 30 Days: Second reminder to IAM DL.
  • 7 Days: Critical alert to IAM DL.
  • < 2 Days: (Optional) Emergency auto-renewal to prevent outage.

Step-by-Step Guide

1

Create Azure Automation Account

Go to the Azure Portal and create a new Automation Account. Enable System Assigned Managed Identity.

2

Assign Permissions

The Managed Identity requires Graph API permissions. Run the following to grant Application.ReadWrite.All (for secrets) and Mail.Send (for alerts).

PowerShell 
$TenantID = "your-tenant-id"
$PrincipalID = "your-managed-identity-object-id"

# Assign Application.ReadWrite.All
New-MgServicePrincipalAppRoleAssignment -ServicePrincipalId $PrincipalID `
-PrincipalId $PrincipalID `
-ResourceId (Get-MgServicePrincipal -Filter "AppId eq '00000003-0000-0000-c000-000000000000'").Id `
-AppRoleId "1bfefb4e-e0b5-418b-a88f-73c46d2cc8e9" 

# Assign Mail.Send (if using Graph for emails)
# AppRoleId for Mail.Send is "b633e1c5-bce6-49a3-97ac-883a5cef74f9"
3

Configure Notification Logic

The runbook logic is designed to reduce noise. It only triggers emails on specific days: 60, 30, and 7 days prior to expiration.

4

Auto-Renew Failsafe

If action hasn't been taken by the time expiration is imminent (e.g., 2 days remaining), the script can auto-generate a new secret to keep the app running, though manual update of the consumer service is still required.

5

Email Delivery

Emails are sent using a shared mailbox via Microsoft Graph or an internal SMTP relay to the IAM Distribution List (e.g., iam-dl@company.com).

Scripts

PowerShell Runbook: Monitor & Alert

This script checks for expiring secrets and sends email notifications to the IAM DL at 60, 30, and 7 day marks.

Runbook.ps1 
try {
    # Connect using Managed Identity
    Connect-MgGraph -Identity
    
    # Configuration
    $AlertDays = @(60, 30, 7)    # Trigger alerts on these days
    $AutoRenewThreshold = 2      # Auto-renew if less than this many days
    $IamDlEmail = "iam-dl@example.com"
    $SenderEmail = "automation-noreply@example.com"
    
    $Apps = Get-MgApplication -All
    
    foreach ($App in $Apps) {
        foreach ($Cred in $App.PasswordCredentials) {
            $ExpiryDate = [DateTime]$Cred.EndDateTime
            $DaysLeft = ($ExpiryDate - (Get-Date)).Days
            
            # --- Alerting Logic (60, 30, 7 Days) ---
            if ($DaysLeft -in $AlertDays) {
                Write-Output "Sending $DaysLeft-day alert for '$($App.DisplayName)' to IAM DL."
                
                $EmailBody = "The client secret for $($App.DisplayName) (App ID: $($App.AppId)) expires in $DaysLeft days on $ExpiryDate. Please rotate credentials."
                
                $Params = @{
                    Message = @{
                        Subject = "ACTION REQUIRED: Secret for $($App.DisplayName) expires in $DaysLeft days"
                        Body = @{ ContentType = "Text"; Content = $EmailBody }
                        ToRecipients = @( @{ EmailAddress = @{ Address = $IamDlEmail } } )
                    }
                }
                # Send-MgUserMail -UserId $SenderEmail -BodyParameter $Params
            }

            # --- Auto-Renewal Failsafe (< 2 Days) ---
            if ($DaysLeft -ge 0 -and $DaysLeft -le $AutoRenewThreshold) {
                Write-Output "CRITICAL: Secret for '$($App.DisplayName)' expires in $DaysLeft days. Auto-renewing..."
                
                $Params = @{ PasswordCredential = @{ DisplayName = "AutoRotate_$(Get-Date -Format 'yyyyMMdd')" } }
                $NewCred = New-MgApplicationPassword -ApplicationId $App.Id -BodyParameter $Params
                
                Write-Output "SUCCESS: New secret created. KeyId: $($NewCred.KeyId)"
                # Add logic here to securely vault the new secret if possible
            }
        }
    }
}
catch {
    Write-Error "An error occurred: $_"
}

Kusto Query (Monitoring)

Track your automation jobs in Azure Monitor.

Query.kql 
AzureDiagnostics 
| where ResourceProvider == "MICROSOFT.AUTOMATION" 
| where Category == "JobStreams" 
| where ResultDescription has "Sending" or ResultDescription has "CRITICAL"
| project TimeGenerated, RunbookName_s, ResultDescription
| order by TimeGenerated desc
